# Windows Hyper-V On/Off — 제품 요구사항 문서 (PRD)

> **버전:** 1.0
> **작성일:** 2026-03-19
> **상태:** 초안

---

## 1. 프로젝트 개요

### 1.1 배경 및 문제 정의

Windows의 Hyper-V 기능은 **WSL2, Docker Desktop, Android 에뮬레이터, Windows Sandbox** 등의 가상화 환경에 필수적이다. 반면, Hyper-V가 활성화되어 있으면 **VirtualBox, VMware** 등 일부 서드파티 하이퍼바이저와 충돌이 발생한다.

현재 사용자가 Hyper-V를 전환하려면:

1. `Windows 기능 켜기/끄기` 패널을 열거나
2. 관리자 PowerShell에서 `bcdedit` / `DISM` 명령을 직접 실행해야 한다.

두 방법 모두 **다단계 클릭** 또는 **명령줄 숙지**를 요구하며, 변경 후 반드시 **재부팅**이 필요하다는 안내도 불분명하다.

### 1.2 솔루션 요약

단일 폼(Single Form) 데스크톱 애플리케이션으로, 현재 Hyper-V 활성화 상태를 즉시 표시하고 **토글 한 번**으로 On/Off 전환 및 재부팅 안내까지 처리한다.

## 4. 기능 요구사항

### 4.1 핵심 기능 (Must Have)

#### FR-01 현재 상태 표시
- 앱 실행 시 **Hyper-V 활성화 여부**를 자동으로 감지한다.
- 감지 방법: `bcdedit /enum {current}` 결과에서 `hypervisorlaunchtype` 값 파싱
- 상태 표현:
  - **ON** → 초록색 레이블 + "Hyper-V 켜짐"
  - **OFF** → 빨간색 레이블 + "Hyper-V 꺼짐"
  - **감지 불가** → 노란색 레이블 + "상태 확인 실패"

#### FR-02 토글 버튼
- 단일 버튼으로 On → Off / Off → On 전환
- 현재 상태에 따라 버튼 레이블 변경:
  - 현재 ON → 버튼 텍스트: "Hyper-V 끄기"
  - 현재 OFF → 버튼 텍스트: "Hyper-V 켜기"

#### FR-03 변경 실행
- **켜기:** `bcdedit /set hypervisorlaunchtype auto`
- **끄기:** `bcdedit /set hypervisorlaunchtype off`
- 명령 실행은 **관리자 권한** 필요 → 미보유 시 UAC 상승 후 재실행

#### FR-04 재부팅 안내
- 변경 성공 후 다이얼로그 표시:
  - 메시지: "변경 사항은 재부팅 후 적용됩니다. 지금 재부팅하시겠습니까?"
  - 버튼: [지금 재부팅] / [나중에]
- [지금 재부팅] 선택 시: `shutdown /r /t 10 /c "Hyper-V 설정 변경 적용"` 실행

#### FR-05 관리자 권한 처리
- 앱 실행 시 현재 프로세스 권한 확인
- 권한 부족 시: UAC 프롬프트를 통해 관리자로 재실행 (`runas`)
- 매니페스트에 `requireAdministrator` 설정 권장

### 4.2 부가 기능 (Nice to Have)

#### FR-06 현재 부팅 상태와 다음 부팅 상태 구분 표시
- `bcdedit`는 **다음 부팅**에 적용될 값을 변경
- 현재 실행 중인 상태(레지스트리 또는 `systeminfo`)와 구분하여 표시

#### FR-07 최소화 트레이 아이콘
- 시스템 트레이에 Hyper-V 상태 아이콘 표시
- 트레이 아이콘 우클릭 → 컨텍스트 메뉴로 빠른 전환

## 8. 유저 스토리 및 수용 기준

### US-01 상태 확인
> 사용자로서, 앱을 실행했을 때 현재 Hyper-V 상태를 즉시 알고 싶다.

**수용 기준:**
- [ ] 앱 실행 후 1초 이내에 ON/OFF 상태가 표시된다.
- [ ] 상태가 색상과 텍스트로 명확히 구분된다.
- [ ] 관리자 권한 없이도 상태 조회는 가능하다.

### US-02 Hyper-V 끄기
> 개발자로서, VirtualBox를 사용하기 위해 Hyper-V를 끄고 싶다.

**수용 기준:**
- [ ] "Hyper-V 끄기" 버튼 클릭 시 UAC 프롬프트가 나타난다(권한 없을 때).
- [ ] 명령 성공 후 재부팅 안내 다이얼로그가 표시된다.
- [ ] [지금 재부팅] 클릭 시 10초 카운트다운 후 재부팅된다.

### US-03 Hyper-V 켜기
> 개발자로서, Docker Desktop을 사용하기 위해 Hyper-V를 다시 켜고 싶다.

**수용 기준:**
- [ ] "Hyper-V 켜기" 버튼 클릭 시 동일한 흐름으로 동작한다.
- [ ] 변경 실패 시 오류 메시지가 표시된다.

### US-04 권한 없이 실행
> 일반 사용자로서, 실수로 관리자 권한 없이 실행해도 앱이 적절히 안내해 주길 원한다.

**수용 기준:**
- [ ] 변경 시도 시 UAC 요청이 발생한다.
- [ ] UAC 거부 시 오류 없이 조용히 종료되거나 안내 메시지를 표시한다.

## 9. 오류 처리

| 오류 상황 | 처리 방법 |
|-----------|-----------|
| bcdedit 실행 실패 | 오류 코드와 함께 메시지 박스 표시 |
| UAC 거부 | 조용히 무시 (사용자가 의도적으로 취소) |
| Hyper-V 미지원 OS | "이 시스템은 Hyper-V를 지원하지 않습니다" 표시 후 버튼 비활성화 |
| 재부팅 명령 실패 | "수동으로 재부팅해 주세요" 안내 |

---

## 10. 제약 사항 및 가정

- Windows Pro / Enterprise / Education 이상에서만 Hyper-V 지원 (Home 에디션 불가)
- `bcdedit` 명령은 관리자 권한 필수
- 변경 사항은 즉시 적용되지 않고 **다음 부팅 시 적용**
- 앱 자체는 설치 불필요한 포터블 EXE로 배포

---

## 11. 향후 고려 사항

- 커맨드라인 인수 지원: `SetHyperV.exe /on`, `/off`, `/status`
- Windows Task Scheduler 연동으로 특정 앱 실행 시 자동 전환
- 설정 파일로 재부팅 자동화 여부 저장
